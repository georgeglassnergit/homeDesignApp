## 2026-07-18 — Pro seam, next row: configurable snapping / constraints (grid · corners · angle-lock)
- **Picked:** the last dormant Pro-seam tier-table row — **"Snapping/constraint settings" (Pro)**. `FEATURE_TIERS['snapping-constraints']` had been defined as `MODE.PRO` since S1 but was never wired to anything, exactly the shape yesterday's `exact-dimensions` was before it was surfaced. Worse, snapping was a *hardcoded fork*: Simple mode grid-snapped at a fixed 0.1 m and **Pro mode used the raw pointer with no snapping at all** — the opposite of what a precision user wants. This slice replaces that fork with three **data-driven** constraints (grid · vertex/corner · angle-lock), keeps Simple's previously-verified defaults byte-identical, and exposes the controls behind the seam so Pro users get real drawing precision. Corner snap also makes drawn rooms close **watertight**, which the CSG openings depend on. *(Branched from the true `origin/main` tip `ccffbf3` = PR #6; no stale-`origin/main` trap this run.)*
- **Changed:**
  - `src/edit/snapping.js` (NEW) — **pure, three-/DOM-free** snapping math: `defaultSnapSettings()` (grid+vertex on, angle off), `normalizeSnapSettings()` (clamps non-positive/out-of-range steps back to safe defaults so bad UI input can't divide-by-zero or loop), `nearestSnapVertex()` (snap onto an existing corner, with an `excludeKeys` set so a dragged corner never snaps to itself), `constrainAngle()` (lock a segment's direction to a multiple of `stepDeg` from its anchor, quantizing length to the grid when grid is on), and `snapPoint()` composing them in priority order **vertex → angle → grid** and returning which constraint fired. Node-testable; importing it under Node is itself the model/view-separation guard.
  - `src/app/state.js` — `createAppState` gains a `snap` settings tree + `setSnap(partial)` (deep-merges one field without restating the others). View-only pref — **never serialized**, so the Phase 1 save format is untouched.
  - `src/edit/tools.js` — `_snap(world, { anchor, excludeKeys })` now runs through `snapPoint` with the live settings instead of the old `mode === simple ? gridSnap : raw` fork. Wall-draw passes the previous corner as the angle anchor; corner-drag passes the grabbed endpoints as `excludeKeys`. Removed the dead `MODE_SIMPLE` gate.
  - `src/main.js` + `index.html` — a **Pro-only** "Snap…" popover (grid on/off + step, snap-to-corners on/off, angle-lock on/off + 15/30/45/90° selector) that edits `app.snap` live and redraws the plan. The `#snap-group` is shown only when `isAvailable('snapping-constraints', mode)` (the single seam gate), so it's invisible in Simple and appears in Pro; click-away closes it. Harness handles `__snap`/`__setSnap`/`__snapSeamVisible`.
